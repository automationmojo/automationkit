Uses a primitive method to create a list of the files and folders in a directory by running the ‘ls -al’ commands on the directory. This is required if the SSH server does not support sftp.

Parameters

  • ssh_client (paramiko.SSHClient) – A paramiko.SSHClient object that has already been connected to the remote server.

  • directory (str) – The remote directory to get a list catalog for.

akit.integration.agents.sshagent.ssh_execute_command(ssh_client: paramiko.client.SSHClient, command: str, pty_params=None, inactivity_timeout: float = 300, inactivity_interval: float = 0.5, chunk_size: int = 1024, attach_pty: bool = False)[source]¶

Runs a command on a remote server using the specified ssh_client. We implement our own version of ssh_execute_command in order to have better control over the timeouts and to make sure all the checks are sequenced properly in order to prevent SSH lockups.

Parameters

  • ssh_client (paramiko.SSHClient) – The paramiko.SSHClient object to utilize when running the command.

  • command (str) – The commandline to run.

  • inactivity_timeout (float) – The timeout for the channel for the ssh transaction.

  • inactivity_interval (float) – The retry interval to wait for the channel to have data ready.

  • chunk_size (int) – The size of the chunks that are read during receive operations

Returns

A tuple with the command result code, the standard output and the standard error output.

Return type

(int, str, str)
