classmethod iteration_advance(iterctx)[source]

The ‘iteration_advance’ API is overridden by derived iterator scope couplings and is called by the sequencer after the scope exits. This class level hook method is used by the sequencer to advance the scope to the next iteration state. The ‘iteration_advance’ API will return a ‘True’ result when the advancement of the iteration state was successful and the scope can be re-entered for execution. The ‘iteration_advance’ API will return a ‘False’ when the advancement of the iteration state has reached the end of its iteration cycle and the scope should not be re-entered.

class akit.testing.testplus.scopecoupling.ScopeCoupling[source]

Bases: akit.coupling.basecoupling.BaseCoupling

The ScopeCoupling object is the base object class that is used for scope declaration. ScopeCoupling derived objects are used to create a hierarchy of scopes that are representative of the scopes of execution that are represented by the runtime environment. These scopes of execution are used to establish the runtime contexts that task and test instantiations can be run inside of. The scopes of a runtime environment are typically hierarchical in nature starting with the root object of a tree and build more complexed environments as they the hierarchy is ascended.

The code of the ScopeCoupling is divided into class level code and instance level code. The hierarchy of the ScopeCoupling derived objects are used by the akit.testing.unittest.testsequencer.Sequencer object to determine the grouping and order of the tests or tasks to be run. The class level code of the ScopeCoupling is run by the akit.sequencer.Sequencer object based on the class hierarchy to establish the order that scopes are entered and exited as the automation sequence is executed by the akit.testing.unittest.testsequencer.Sequencer object. The ScopeCoupling class level code, is executed before any object that inherits from a ScopeCoupling derived object is instantiated, so the state for the scope has been establish.

The ScopeCoupling instance level code is utilized to inter-operate with the state of the scope and also provides scope specific functionality.

..notes :

A scope represents a predefined state that is reached by the execution of code. The state represents a requirement that is needed to be met in order for a task to be able to run.

Scopes have a name that is like a file system path /environment/configuration

Scopes can contain state and they are deposited into the context in a leaf just like other nodes.

akit.testing.testplus.testref module

class akit.testing.testplus.testref.TestRef(testfunc: function)[source]

Bases: object

The TestRef objects are used to refer to a reference to a test. We use TestRef instances to reference the tests that are going to be run so we can control the lifespan of test case instances and control our memory consumption during test runs with large collections of test cases.

The TestRef object allows us to delay the creation of test runtime instance data and state until it is necessary to instantiate it and allows us to cleanup the runtime instance and state as soon as it is no longer being used.
